This report presents \textbf{InlinePDF}, a local-first PDF toolkit designed to process files directly in the browser instead of uploading documents to remote servers. The project addresses a common gap in traditional client-server PDF tools: users often trade privacy for convenience. InlinePDF focuses on three goals: data minimization, practical performance for everyday document workflows, and transparent implementation through open-source development. The report documents the problem context, system design, technology stack, testing approach, current limits, and future roadmap.
